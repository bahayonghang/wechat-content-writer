# 写作风格指南

本指南详细说明了插件推荐的内容创作风格，确保生成的文章专业、客观、有价值。

## 核心原则

### 客观中立

- ✅ 使用第三人称视角
- ✅ 基于数据和事实
- ✅ 平衡呈现不同观点
- ❌ 避免主观臆断
- ❌ 避免情绪化表达

### 准确专业

- ✅ 技术术语准确
- ✅ 数据来源可追溯
- ✅ 引用规范
- ❌ 避免过度简化
- ❌ 避免误导性表述

### 完整全面

- ✅ 包含局限性说明
- ✅ 提及挑战和风险
- ✅ 避免片面乐观
- ❌ 不夸大效果
- ❌ 不隐瞒问题

## 标题规范

### 推荐标题

- ✅ "基于多模态融合的FeO浓度实时预测技术研究"
- ✅ "深度学习在医疗影像诊断中的应用进展"
- ✅ "2024年大语言模型技术发展报告"

### 避免标题

- ❌ "震惊！AI竟有这样的能力！"
- ❌ "深度学习：从入门��精通（附代码）"
- ❌ "必看！这个黑科技太强大了！"

### 标题结构建议

```
[研究主题] + [技术/方法] + [研究性质]
```

示例：
- "多模态大语言模型（研究主题）+ 视觉-语言对齐技术（技术）+ 研究综述（性质）"
- "强化学习（研究主题）+ 在机器人控制中的应用（技术）+ 案例分析（性质）"

## 文章结构

### 学术类文章结构

```markdown
## 摘要
简要概括研究内容、方法和结论

## 1. 引言
- 研究背景
- 问题定义
- 研究意义

## 2. 相关工作
- 研究现状
- 主要方法
- 存在问题

## 3. 方法/技术
- 核心思想
- 技术架构
- 关键算法

## 4. 实验与分析
- 实验设置
- 结果分析
- 对比评估

## 5. 结论
- 主要贡献
- 局限性
- 未来方向

## 参考文献
```


## 数据引用规范

### 引用格式

```
根据 [来源] 的研究，[结论]。

[作者等] ([年份]) 指出，[结论]。

数据显示，[具体数据] [来源]。
```

### 示例

```
# 正确
根据 Vaswani 等人 (2017) 的研究，Transformer 架构在机器翻译任务上取得了显著效果。

实验数据显示，该模型在 WMT 2014 英德翻译任务上达到 28.4 BLEU 值 (Vaswani et al., 2017)。

# 避免
研究表明效果很好。（数据不明确）
论文说效果惊人。（表述主观）
```

## 局限性声明

### 为什么重要

客观的技术分析必须包含局限性说明，这体现专业性，也帮助读者全面理解。

### 如何表述

```markdown
## 局限性

本研究/技术存在以下局限：

1. **数据规模**：实验数据量有限，结论普适性需进一步验证
2. **计算成本**：模型推理成本较高，实际部署存在挑战
3. **泛化能力**：在特定场景表现良好，跨领域适应性待研究

## 未来工作

未来可从以下方向改进：

1. 扩大数据规模，验证结论普适性
2. 优化模型结构，降低计算成本
3. 探索跨领域迁移方法
```

## 对比示例

### 修改前（AI味重）

```markdown
## 震惊！GPT-4 强大到令人窒息！

GPT-4 来了，这款黑科技简直是神器！笔者试用后发现，它的能力简直突破了天际！

### 火眼金睛般的视觉能力

GPT-4 能秒看图片，简直是开了挂！不管是什么图，它都能一眼看穿，准确率达到恐怖的99%！

### 完美的代码能力

写代码？小菜一碟！GPT-4 分分钟给你写出完美的代码，简直是想什么有什么！

这绝对是颠覆性的技术，必将彻底改变世界！
```

### 修改后（客观专业）

```markdown
## GPT-4 技术能力分析

GPT-4 是 OpenAI 发布的多模态大语言模型，在多项任务上表现出显著提升。

### 视觉理解能力

GPT-4 引入了视觉处理能力，能够理解图像内容并与文本进行交互。实验数据显示，在图像描述任务上，模型表现出较强的场景理解能力。

### 代码生成能力

在编程任务中，GPT-4 能够根据需求生成代码片段。根据公开评测，模型在 Python 编程测试中达到 67.0% 的准确率。

### 局限性

尽管性能提升显著，但模型仍存在以下局限：
- 幻觉问题未完全解决
- 对上下文长度有限制
- 推理成本较高

随着技术发展，这些问题有望得到改善。
```
