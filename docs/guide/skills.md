# 技能使用

插件包含 3 个自动技能，根据关键词自动触发，提供更自然的使用体验。

## 技能概览

| 技能 | 触发关键词 | 用途 | 输出风格 |
|------|-----------|------|----------|
| `literature-research` | "搜索文献"、"literature search" | 学术文献搜索与研究 | 专业研究导向 |
| `pdf-analysis` | "解析PDF"、"analyze PDF" | PDF文档分析与解读 | 通俗易懂 |
| `pdf-analysis-objective` | "客观分析"、"专业解读" | PDF客观技术分析 | 严谨专业 |

---

## literature-research

自动学术文献搜索技能，帮助快速找到相关研究资料。

### 触发方式

以下表述会自动触发此技能：

- "搜索文献：[主题]"
- "查找学术资料：[主题]"
- "literature search: [topic]"
- "latest research: [topic]"

### 功能特点

- 多源搜索：同时搜索 arXiv、Google Scholar
- 智能过滤：按相关性、时间筛选
- 结果汇总：生成结构化的文献综述
- 引用格式：自动生成标准引用

### 使用示例

```
# 中文触发
搜索文献：多模态大语言模型
查找学术资料：深度学习在计算机视觉中的应用

# 英文触发
literature search: multimodal large language models
latest research: diffusion models
```

### 输出内容

技能将返回：

1. **文献列表**
   - 标题与作者
   - 发表时间
   - 来源（arXiv/Scholar）
   - 引用链接

2. **研究摘要**
   - 主要研究方向
   - 技术方法概要
   - 实验结果

3. **研究趋势**
   - 近期热点
   - 发展方向

### 示例输出

```markdown
## 文献搜索结果：多模态大语言模型

### 找到 15 篇相关文献

#### 1. "Flamingo: a Visual Language Model for Few-Shot Learning"
- **作者**: Jean-Baptiste Alayrac et al.
- **发表**: 2022-11-03
- **来源**: arXiv
- **链接**: https://arxiv.org/abs/2211.09611

**摘要**: 本文提出 Flamingo 模型...

#### 2. "BLIP-2: Bootstrapping Language-Image Pre-training..."
- **作者**: Junnan Li et al.
- **发表**: 2023-01-17
- **来源**: arXiv
- **链接**: https://arxiv.org/abs/2301.12597

**摘要**: 本文介绍 BLIP-2...

### 研究趋势分析
- **主要方向**: 视觉-语言对齐、高效参数适配
- **热门方法**: Q-Former、Adapter、PeFT
- **应用场景**: 图像描述、VQA、视觉推理
```

## pdf-analysis

PDF 文档分析技能，生��通俗易懂的解读内容。

### 触发方式

以下表述会自动触发此技能：

- "解析PDF：[文件]"
- "解读文档：[文件]"
- "analyze PDF: [file]"
- "解读这份论文"

### 功能特点

- 自动提取关键信息
- 生成通俗解读
- 突出重点内容
- 添加解释说明

### 使用示例

```
# 上传 PDF 文件后
解析PDF：[上传的文件]

# 或直接描述
解读这份论文的内容
```

### 输出风格

此技能采用**通俗易懂**的风格，适合：
- 技术科普文章
- 入门教程内容
- 面向大众的解读

### 示例输出

```markdown
## PDF 解读：GPT-4 技术报告

### 这篇论文讲了什么？

简单来说，GPT-4 是一个超大的语言模型，它能看图、能写代码、能回答各种复杂问题。

### 核心亮点

1. **多模态能力**：不仅能处理文字，还能看懂图片
2. **规模巨大**：参数量比 GPT-3 大很多
3. **性能强悍**：在各种考试中表现接近人类专家水平

### 技术原理（用人话说）

想象 GPT-4 是一个读过上万亿页书的超级大脑...
```

## pdf-analysis-objective

PDF 客观技术分析技能，生成专业严谨的解读。

### 触发方式

以下表述会自动触发此技能：

- "客观分析：[文件]"
- "专业解读：[文件]"
- "技术分析：[主题]"

### 功能特点

- 第三人称客观表述
- 技术术语准确
- 包含局限性说明
- 数据驱动分析

### 使用示例

```
# 上传 PDF 文件后
客观分析：[上传的文件]

# 或直接描述
专业解读：这份论文的技术贡献
```

### 输出风格

此技能采用**严谨专业**的风格，适合：
- 学术论文解读
- 技术深度分析
- 专业研究报告

### 示例输出

```markdown
## 技术分析：GPT-4 系统架构

### 概述

GPT-4 是一种基于 Transformer 架构的多模态大语言模型，支持文本和图像输入。

### 技术架构

#### 模型结构
- 基础架构：Transformer decoder-only
- 参数规模：未公开具体数值（推测超 1T）
- 训练数据：混合文本与图像数据

#### 多模态处理
- 图像编码：独立视觉编码器
- 特征对齐：交叉注意力机制
- 输出融合：统一文本输出空间

### 性能评估

| 任务 | GPT-4 | GPT-3.5 |
|------|-------|---------|
| MMLU | 86.4% | 70.0% |
| Python编程 | 67.0% | 48.1% |

### 局限性

- 幻觉问题仍然存在
- 推理成本较高
- 对上下文长度有限制

### 结论

GPT-4 在多模态理解和复杂推理方面表现出显著提升，但仍需解决可靠性和成本问题。
```

## 工作流建议

### 学术文章创作流程

```
1. 搜索文献
   → "搜索文献：深度学习在NLP中的应用"

2. 客观分析 PDF
   → 上传 PDF + "客观分析：这份论文"

3. 生成文章
   → /create-paper "XXX研究综述" --category 文献解读
```

### 技术分析文章流程

```
1. 文献调研
   → "literature search: attention mechanisms"

2. 技术分析
   → 上传 PDF + "专业解读：架构设计"

3. 深度文章
   → /create-tech "XXX架构解析" --category 技术分享 --aspect architecture
```

### 科普文章流程

```
1. 了解主题
   → "搜索文献：XXX技术"

2. 通俗解读
   → 上传 PDF + "解析PDF：这份论文"

3. 教程文章
   → /create-tutorial "XXX入门教程" --category 技术分享 --level beginner
```

## 技能与命令配合

技能和命令可以灵活配合使用：

### 先技能后命令

```
# 步骤1：搜索文献
搜索文献：多模态大模型

# 步骤2：选择论文并分析
[上传PDF] + 客观分析：这篇论文

# 步骤3：生成文章
/create-paper "多模态大模型研究综述" --category 文献解读
```

### 技能独立使用

```
# 仅搜索文献，不生成文章
搜索文献：最新的强化学习研究

# 仅分析 PDF，不生成文章
[上传PDF] + 解析这份论文
```

### 命令独立使用

```
# 手动提供内容，不使用技能
/create-article-objective "自定义主题" --category 技术分享 --source manual
```
